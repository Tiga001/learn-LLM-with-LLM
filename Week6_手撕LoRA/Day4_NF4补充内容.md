# 深入浅出：QLoRA 中 NF4 量化的数学原理与计算推导

在深度学习模型（特别是大语言模型）的量化中，QLoRA 提出了一种极具创新性的数据类型：**NF4 (4-bit NormalFloat)**。它之所以能在 4-bit 极低精度下保持模型性能，核心在于它是一把“**为正态分布量身定制的非均匀刻度尺**”。

以下我们将从最硬核的底层数学推导开始，手摇计算器算出这把尺子的刻度，再演示这把尺子在实际模型中是怎么用的。

---

## 零、硬核拆解：NF4 魔法字典的具体数值推导

神经网络的权重通常服从以 0 为中心的标准正态分布 $W \sim N(0, 1)$。NF4 的核心数学思想是：**利用标准正态分布的逆累积分布函数（分位数函数） $\Phi^{-1}(p)$，根据概率密度来划分刻度，让每个量化区间内包含相同数量的权重参数。**

为了得到这 16 个固定的刻度（对应 4-bit 的 16 个状态），其详细推导步骤如下：

### 第 1 步：确定 16 个不对称的概率采样点 ($p$)
为了让这 16 个数完美覆盖正态分布，我们需要在 0 到 1 的概率区间内取 16 个 $p$ 值。如果绝对均匀地取，0 会卡在中间取不到。为了**强制保留一个完美的 0（用于处理 Padding 和稀疏性）**，QLoRA 采用了非对称分配法：
* **1 个零点**：固定概率 $p = 0.5$。
* **7 个负数点**：在 $p < 0.5$ 的区间内取 7 个点。
* **8 个正数点**：在 $p > 0.5$ 的区间内取 8 个点。
*(注：为防止 $\Phi^{-1}$ 在两端出现无穷大，设定了微小的截断偏移量，所有概率取样都在 0.0323 到 0.9677 之间进行。)*

### 第 2 步：计算原始的物理坐标 ($Z$ 值)
拿到这 16 个概率 $p_i$ 后，我们把它们全部喂给逆累积分布函数，算出它们在标准正态分布上的真实横坐标 $Z_i$：
$$Z_i = \Phi^{-1}(p_i)$$

我们挑三个最具代表性的点来计算：
* **最小的负数（第 1 个点）：** $Z_1 = \Phi^{-1}(0.0323) \approx -1.8477$
* **绝对零点（第 8 个点）：** $Z_8 = \Phi^{-1}(0.5000) = 0.0000$
* **最大的正数（第 16 个点）：** $Z_{16} = \Phi^{-1}(0.9677) \approx 1.8477$

### 第 3 步：全局极值归一化 (Normalization)
为了让字典变得通用，符合 $[-1, 1]$ 的标准计算范围，我们需要把算出所有 $Z_i$ 都除以绝对值的最大值 $Z_{max}$（即 1.8477）：
$$q_i = \frac{Z_i}{1.8477}$$

代入刚才的三个点：
* 最小值：$q_1 = \frac{-1.8477}{1.8477} = -1.0000$
* 零点：$q_8 = \frac{0.0000}{1.8477} = 0.0000$
* 最大值：$q_{16} = \frac{1.8477}{1.8477} = 1.0000$

### 第 4 步：生成最终的 NF4 字典表
对所有 16 个点完成上述计算后，我们就得到了写死在 QLoRA 源码中的 **16 个 NF4 标准常量**（保留四位小数），记作字典 $Q_{NF4}$：
* **7 个负数：** `-1.0000`, `-0.6962`, `-0.5251`, `-0.3949`, `-0.2844`, `-0.1848`, `-0.0911`
* **1 个零：** `0.0000`
* **8 个正数：** `0.0796`, `0.1609`, `0.2461`, `0.3379`, `0.4407`, `0.5626`, `0.7230`, `1.0000`

## 一、NF4 量化的具体计算过程演示

了解了这把“非均匀刻度尺”是怎么来的之后，我们看看当模型运行量化时，具体的数学计算过程。

假设神经网络中有一小块权重（为了方便演示，我们只取 4 个数字，实际中通常是 64 个为一组进行分块）：
$$W = [2.05, -1.10, 0.50, -0.10]$$

### 第 1 步：寻找缩放因子 (Scale Factor)
找出这组数字里**绝对值最大**的数，作为缩放因子 $c$。
$$c = \max(|2.05|, |-1.10|, |0.50|, |-0.10|) = 2.05$$

### 第 2 步：归一化 (Normalization)
把原始权重 $W$ 里的每一个数字，都除以缩放因子 $c$，缩小到 $[-1, 1]$ 之间。
$$W_{norm} = \frac{W}{2.05} = [1.000, -0.536, 0.244, -0.048]$$

### 第 3 步：量化 (Quantization) —— 找最近的刻度
拿着缩小后的 $W_{norm}$，去刚才推导出的 NF4 字典表 $Q_{NF4}$ 上找离它们**欧氏距离最近**的刻度，并记录下对应的**状态索引（0-15）**。这就是模型量化误差的来源。

* **1.000** $\rightarrow$ 完美命中字典里的 $1.000$，索引为 **15**。
* **-0.536** $\rightarrow$ 离字典里的 $-0.5251$ 最近，索引为 **2**。
* **0.244** $\rightarrow$ 离字典里的 $0.2461$ 最近，索引为 **10**。
* **-0.048** $\rightarrow$ 离字典里的 $0.0000$ 最近，索引为 **7**。

**【落盘存储状态】**
此时，显存和硬盘里不再存储庞大的浮点数，而是只存：
1. 4-bit 的整数索引矩阵：`[15, 2, 10, 7]`
2. 这一块的高精度缩放因子：$c = 2.05$

## 二、反量化 (Dequantization) 与误差分析

当模型进行推理计算时，它会进行光速“解压缩”还原数值。

### 第 4 步：反量化还原
1. **查表**：根据索引 `[15, 2, 10, 7]`，从 $Q_{NF4}$ 字典中掏出对应的浮点数：$[1.000, -0.5251, 0.2461, 0.000]$。
2. **乘回缩放因子**：把这些数字乘回 $c = 2.05$。

还原后的近似权重矩阵：
$$W_{approx} = [1.000 \times 2.05,\ -0.5251 \times 2.05,\ 0.2461 \times 2.05,\ 0.000 \times 2.05]$$
$$W_{approx} = [2.050, -1.076, 0.504, 0.000]$$

### 误差大盘点

我们将**原始权重**和**还原后的权重**放在一起对比：

| 原始权重 $W$ | 还原权重 $W_{approx}$ | 误差 (Absolute Error) |
| :--- | :--- | :--- |
| **2.05** | **2.050** | **0** (完美还原) |
| **-1.10** | **-1.076** | **0.024** |
| **0.50** | **0.504** | **0.004** |
| **-0.10** | **0.000** | **0.100** |

这就是量化的真实代价。由于底层权重使用了吻合其概率分布的 NF4，这种误差被降到了极低；再配合上外挂的、未量化的高精度 LoRA 适配器进行微调，模型能够完美地“学习并抵消”掉这部分误差，从而在极少显存占用的情况下实现近乎全参数微调的性能。
