## 一、LoRA 论文精读

### 1.1 论文基本信息

- **标题**：*LoRA: Low-Rank Adaptation of Large Language Models*
- **作者**：Edward J. Hu, Yelong Shen, Phillip Wallis, Zeyuan Allen-Zhu, Yuanzhi Li, Shean Wang, Lu Wang, Weizhu Chen
- **机构**：Microsoft Research
- **时间**：2021 年 6 月（arXiv），ICLR 2022
- **核心贡献**：提出用低秩矩阵分解来近似微调时的权重更新，实现参数高效微调且推理零开销

### 1.2 动机：全参数微调的痛点

论文开篇指出了全参数微调（Full Fine-tuning）面临的核心困境：

```
困境 1: 存储成本
  GPT-3 175B → 每个下游任务需要存储一份完整模型
  10 个任务 = 10 × 350 GB = 3.5 TB
  → 部署极不经济

困境 2: 部署灵活性
  多租户场景需要快速切换不同任务的模型
  全参数微调 → 每次切换都要加载完整模型
  → 延迟高、资源浪费

困境 3: 训练成本
  175B 模型的全参数微调需要数百张 GPU
  → 大多数研究者和企业无法负担
```

### 1.3 核心洞察：Intrinsic Dimensionality

LoRA 的理论基础来自 Aghajanyan et al. (2021) 的一项关键发现：

> **预训练语言模型具有很低的"内在维度"（Intrinsic Dimensionality）**。即使随机投影到一个低维子空间中进行优化，模型仍然可以有效地学习。

```
实验发现 (Aghajanyan et al., 2021):
  RoBERTa-base (125M 参数):
    全参数微调: 更新 125M 个参数 → 效果 = 基线
    随机子空间 200 维: 更新 200 个参数 → 效果达到基线的 90%!

  GPT-3 (175B 参数):
    在 MRPC 任务上，内在维度仅为 ~200
    → 175B 参数的调整只需要在 ~200 维子空间中完成

启示:
  Δθ = θ_SFT - θ_pretrain 是一个高维向量
  但它的"有效维度"非常低
  → 可以用低秩矩阵来近似！
```

### 1.4 LoRA 方法

#### 核心思想

对于预训练权重矩阵 $W_0 \in \mathbb{R}^{d \times k}$，LoRA 不直接更新 $W_0$，而是学习一个**低秩增量** $\Delta W$：

$$W = W_0 + \Delta W = W_0 + BA$$

其中：
- $B \in \mathbb{R}^{d \times r}$，$A \in \mathbb{R}^{r \times k}$
- $r \ll \min(d, k)$ 是低秩分解的秩
- $W_0$ 冻结不更新，只训练 $A$ 和 $B$

#### 前向传播

$$h = W_0 x + \Delta W x = W_0 x + BAx$$

在实际实现中，LoRA 还引入了一个缩放因子 $\frac{\alpha}{r}$：

$$h = W_0 x + \frac{\alpha}{r} BAx$$

其中 $\alpha$ 是一个常数超参数，通常设为 $r$ 的整数倍。

```
┌────────────────────────────────────────────────────┐
│  LoRA 前向传播:                                      │
│                                                      │
│            x (输入)                                   │
│            │                                         │
│     ┌──────┴──────┐                                  │
│     │             │                                  │
│     ▼             ▼                                  │
│    W₀x          A: r×k                              │
│  (冻结❄️)        │                                   │
│     │            ▼                                   │
│     │          B: d×r                                │
│     │            │                                   │
│     │            ▼                                   │
│     │         (α/r)·BAx                              │
│     │            │                                   │
│     └──────┬─────┘                                   │
│            ▼                                         │
│          h = W₀x + (α/r)·BAx                        │
│                                                      │
│  训练: 只更新 A 和 B (可训练 🔥)                      │
│  推理: W' = W₀ + (α/r)·BA，直接用 W'x               │
└────────────────────────────────────────────────────┘
```

#### 初始化策略

- $A$：用 Kaiming 均匀初始化（或高斯随机初始化）
- $B$：初始化为**全零矩阵**

$$\Delta W = BA = \mathbf{0} \cdot A = \mathbf{0}$$

这保证了训练开始时 LoRA 分支的输出为零，模型行为完全等同于预训练模型。

#### 应用位置

论文在 Transformer 中测试了不同权重矩阵应用 LoRA 的效果：

| 应用位置 | $W_q$ | $W_k$ | $W_v$ | $W_o$ | $W_{\text{up}}$ | $W_{\text{down}}$ |
|---------|:-----:|:-----:|:-----:|:-----:|:-----:|:-----:|
| 论文默认 | ✅ | ❌ | ✅ | ❌ | ❌ | ❌ |
| 常见实践 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |

论文发现**在 $W_q$ 和 $W_v$ 上同时应用 LoRA** 效果最好（在相同参数预算下优于只在单个矩阵上应用）。

### 1.5 关键实验结果

#### 实验 1：GPT-3 175B 上的效果

| 方法 | 可训练参数 | WikiSQL | SAMSum | MNLI-m |
|------|---------|---------|--------|--------|
| Full FT | 175B | 73.8 | — | — |
| BitFit | 14.2M | 71.3 | — | — |
| Adapter (H) | 7.1M | 73.2 | — | — |
| **LoRA** | **4.7M** | **73.4** | — | — |

**关键发现**：LoRA 用 0.003% 的参数就达到了接近甚至超过全参数微调的效果。

#### 实验 2：不同 rank 的影响

| rank $r$ | 可训练参数 | WikiSQL Acc |
|---------|---------|------------|
| 1 | 0.3M | 68.8 |
| 2 | 0.6M | 72.4 |
| 4 | 1.2M | 73.0 |
| 8 | 2.4M | 73.2 |
| 64 | 18.9M | 73.4 |
| 256 | 75.5M | 73.2 |

**关键发现**：
1. **非常小的 rank（如 4-8）就足够了**，增大 rank 收益递减
2. 这进一步验证了"微调权重更新是低秩的"假说
3. rank=256 反而略有下降，可能因为过拟合

#### 实验 3：权重矩阵低秩性分析

论文对 GPT-3 的 $\Delta W$ 做了 SVD 分析，发现：

```
ΔW 的奇异值分布:
  前几个奇异值: ████████████████████████  >> 其余
  第 4-8 个:    ████                       明显下降
  第 8+ 个:     ▏                          接近零

结论:
  ΔW 的有效秩远小于其矩阵维度
  rank=4 就能捕获 ΔW 的大部分信息
  → 低秩假设在实践中成立
```

### 1.6 LoRA 的核心优势总结

| 优势 | 说明 |
|------|------|
| **推理零开销** | 训练后将 $BA$ 合并到 $W_0$，推理与原始模型相同 |
| **参数高效** | 仅需 0.01%~0.1% 的可训练参数 |
| **存储高效** | 每个任务只需保存低秩矩阵（几 MB） |
| **切换灵活** | 多任务只需切换轻量 LoRA 权重 |
| **无额外推理延迟** | 与 Adapter / Prefix Tuning 形成鲜明对比 |
| **与优化器解耦** | 优化器状态只需覆盖 LoRA 参数 |

## 二、QLoRA 论文精读

### 2.1 论文基本信息

- **标题**：*QLoRA: Efficient Finetuning of Quantized LLMs*
- **作者**：Tim Dettmers, Artidoro Pagnoni, Ari Holtzman, Luke Zettlemoyer
- **机构**：University of Washington
- **时间**：2023 年 5 月
- **核心贡献**：通过三大创新（NF4 量化 + 双重量化 + 分页优化器），让单张 48GB GPU 也能微调 65B 模型，且效果接近 16-bit 全参数微调

### 2.2 动机：LoRA 还不够省

即使用了 LoRA，微调大模型的显存瓶颈仍然在于**基座模型本身**：

```
LoRA 微调 LLaMA-65B 的显存分析:

  基座模型 (FP16):      ~130 GB  ← 瓶颈在这里！
  LoRA 参数 (r=16):     ~33 MB
  优化器状态:            ~132 MB
  梯度:                 ~33 MB
  激活值:               ~20 GB
  ──────────────────────────
  总计:                 ~150 GB → 需要 2×A100 80GB

问题: 虽然可训练参数很少，但基座模型必须以 FP16 加载
解决思路: 能否用更低精度存储基座模型？
```

### 2.3 QLoRA 的三大创新

#### 创新 1：NF4（4-bit NormalFloat）量化

传统的 4-bit 量化（INT4/FP4）在量化神经网络权重时信息损失较大。Dettmers 观察到预训练模型的权重分布接近**正态分布**，提出了 NF4 量化方案：

```
核心思想:
  权重分布 ≈ 正态分布 N(0, σ²)
  → 量化点应该按正态分布的分位数来设置
  → 让每个量化区间包含大约相同比例的权重值

INT4 均匀量化 (16 个等间距点):
  量化点: [-8, -7, ..., -1, 0, 1, ..., 7]
  问题: 权重集中在 0 附近，但量化点是均匀分布的
  → 0 附近分辨率不够，远离 0 的区间浪费了

NF4 量化 (16 个按分位数设置的点):
  量化点: [-1.0, -0.6962, -0.5251, -0.3949, -0.2844,
           -0.1848, -0.0911, 0.0,
            0.0796, 0.1609, 0.2461, 0.3379, 0.4407,
            0.5626, 0.7230, 1.0]
  → 0 附近点更密集（因为权重集中在这里）
  → 每个量化区间内的权重数量近似相等
  → 信息论最优！
```

NF4 的数学构造过程：

1. 对标准正态分布 $\mathcal{N}(0, 1)$ 计算 $2^b + 1$ 个等概率分位数
2. 将这些分位数归一化到 $[-1, 1]$
3. 量化时先将权重除以其 absmax 归一化到 $[-1, 1]$，再找最近的 NF4 量化点

#### 创新 2：双重量化（Double Quantization）

量化需要为每个量化块（block）存储一个缩放因子（scale）。对于 4-bit 量化，这些 FP32 的缩放因子本身也占不少空间：

```
单重量化:
  每 64 个权重一个量化块
  每个块: 64 × 4 bit (权重) + 32 bit (缩放因子)
  平均每个权重: 4 + 32/64 = 4.5 bit

双重量化:
  第一层: 权重 → 4-bit (NF4)，每 64 个一组，产生 FP32 缩放因子
  第二层: 将 FP32 缩放因子量化为 FP8，每 256 个缩放因子一组
  
  平均每个权重:
    权重本身: 4 bit
    第一层缩放因子: 8/64 = 0.125 bit
    第二层缩放因子: 32/(64×256) = 0.002 bit
    总计: ≈ 4.127 bit

节省: 4.5 bit → 4.127 bit → 每个参数节省 0.373 bit
  对 65B 模型: 节省 ~3 GB 显存
```

#### 创新 3：分页优化器（Paged Optimizers）

利用 NVIDIA 统一内存（Unified Memory）特性，在 GPU 显存不够时自动将优化器状态卸载到 CPU 内存：

```
问题场景:
  长序列训练时，激活值可能产生显存尖峰
  → GPU 显存不够 → OOM 崩溃

分页优化器:
  正常训练时: 优化器状态在 GPU 上
  显存不够时: 自动将部分状态转移到 CPU
  需要时:     自动转回 GPU
  
  类比: 操作系统的虚拟内存分页机制
  → 用 CPU 内存作为 GPU 显存的"交换分区"
  → 几乎不影响训练速度（GPU 间歇性空闲时做传输）
```

### 2.4 QLoRA 完整训练流程

```
┌──────────────────────────────────────────────────────┐
│  QLoRA 训练流程:                                       │
│                                                        │
│  1. 加载预训练模型并量化为 NF4                           │
│     LLaMA-65B (FP16: 130GB) → NF4 (4-bit: ~33GB)     │
│                                                        │
│  2. 在冻结的 NF4 模型上注入 LoRA 适配器                  │
│     LoRA 参数以 BF16 精度训练                           │
│                                                        │
│  3. 前向传播                                            │
│     NF4 权重 → 反量化为 BF16 → 计算 W₀x               │
│     LoRA 权重 (BF16) → 计算 BAx                        │
│     h = W₀x + (α/r)·BAx                               │
│                                                        │
│  4. 反向传播                                            │
│     梯度只通过 LoRA 参数（A, B）回传                     │
│     NF4 权重不接收梯度（冻结）                           │
│                                                        │
│  5. 优化器更新                                          │
│     只更新 LoRA 参数                                    │
│     使用分页 AdamW 优化器                               │
│                                                        │
│  6. 训练完成后                                          │
│     可选: 将 LoRA 合并到反量化后的模型中                 │
│     或: 保持 NF4 + LoRA 的部署方式                      │
└──────────────────────────────────────────────────────┘
```

### 2.5 QLoRA 关键实验结果

#### 实验 1：量化精度对微调效果的影响

| 基座模型精度 | 微调方法 | MMLU 5-shot |
|------------|---------|------------|
| Float16 | Full FT | 63.4 |
| Float16 | LoRA | 63.4 |
| **NF4** | **QLoRA** | **63.9** |
| FP4 | QLoRA | 63.2 |
| INT4 | QLoRA | 62.5 |

**关键发现**：
1. NF4 + QLoRA **不逊于** 16-bit 全参数微调
2. NF4 显著优于 FP4 和 INT4（信息论最优量化的价值）
3. 量化带来的信息损失可以被 LoRA 的微调"补偿"

#### 实验 2：显存效率

| 模型 | Full FT (FP16) | LoRA (FP16) | QLoRA (NF4) |
|------|---------------|-------------|-------------|
| LLaMA-7B | ~88 GB | ~21 GB | **~10 GB** |
| LLaMA-13B | ~160 GB | ~37 GB | **~18 GB** |
| LLaMA-33B | ~400 GB | ~90 GB | **~42 GB** |
| LLaMA-65B | ~780 GB | ~170 GB | **~48 GB** |

**革命性意义**：QLoRA 让 **单张 48GB A100** 就能微调 65B 模型。

#### 实验 3：Guanaco 模型

QLoRA 论文基于自己的方法训练了 **Guanaco** 系列模型：

```
Guanaco 成绩:
  Guanaco-65B: 在 Vicuna 排行榜上达到 ChatGPT 的 99.3% 水平
  训练成本: 单张 GPU + 24 小时
  
  对比:
  ChatGPT: 数百~数千张 GPU + 数周训练
  Guanaco: 1 张 GPU + 1 天训练 → 效果接近
  
  → 微调的民主化：几乎人人都可以训练高质量对话模型
```

## 三、LoRA 在 PEFT 方法中的定位

### 3.1 PEFT 方法演进总览

```
2019  Adapter — 在每层插入瓶颈网络
  │   优点: 效果好    缺点: 推理有额外延迟
  ▼
2021  Prompt / Prefix Tuning — 添加可训练 token/前缀
  │   优点: 参数极少  缺点: 小模型效果差，占序列长度
  ▼
2021  LoRA — 低秩分解权重更新 ← 当前主流
  │   优点: 推理零开销、效果好、实现简洁
  ▼
2023  QLoRA — 量化 + LoRA
  │   优点: 极致显存效率  缺点: 训练略慢
  ▼
2024  DoRA / LoRA+ / GaLore — 持续改进
      DoRA: 将 LoRA 分解为方向和幅值
      LoRA+: 对 A 和 B 使用不同学习率
      GaLore: 梯度低秩投影
```

### 3.2 为什么 LoRA 成为事实标准？

```
1. 推理零开销
   LoRA 是唯一能在推理时完全消除额外计算的 PEFT 方法
   W' = W₀ + BA → 直接用 W'，和原始模型一样
   
2. 效果接近全参数微调
   在绝大多数 benchmark 上差距 < 1%
   
3. 实现极其简洁
   核心代码不到 50 行
   HuggingFace PEFT 库提供开箱即用的支持
   
4. 生态支持完善
   PEFT 库、TRL（RLHF 训练）、LLaMA-Factory 等都以 LoRA 为核心
   
5. 与量化完美结合
   QLoRA = NF4 + LoRA → 单卡微调大模型成为可能
   → 极大降低了 LLM 微调的门槛
```

## 五、从 LoRA 到 DoRA：最新演进（选读）

### 5.1 DoRA (Weight-Decomposed Low-Rank Adaptation)

DoRA (2024) 将权重分解为**幅值**和**方向**两个分量，只对方向分量应用 LoRA：

$$W' = m \cdot \frac{W_0 + BA}{\|W_0 + BA\|_c}$$

其中 $m$ 是可学习的幅值向量，$\|\cdot\|_c$ 是列归一化。

直觉：Full FT 同时调整权重的幅值和方向，而 LoRA 将两者耦合在一起。DoRA 解耦后能更精准地模拟 Full FT 的行为。

### 5.2 LoRA+

LoRA+ (2024) 的核心发现：$A$ 和 $B$ 矩阵应该使用**不同的学习率**。

$$\text{lr}_B = \lambda \cdot \text{lr}_A, \quad \lambda \gg 1$$

论文建议 $\lambda = 16$，即 $B$ 的学习率是 $A$ 的 16 倍。

理论解释：在训练初期，$A$ 承担特征提取的角色（需要精细调整），$B$ 承担投射回高维空间的角色（可以大步更新）。
