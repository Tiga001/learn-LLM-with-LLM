## Part 2：综合项目目标

### 2.1 项目定义

本周建议的综合项目是：

> 构建一个带知识库检索能力、支持多轮对话、并可按需调用工具的课程问答 Agent。

它至少支持：

- 多轮对话
- 知识库检索
- 一个或多个工具调用
- 对答案来源的基本约束

### 2.2 典型使用场景

```text
用户: LoRA 和 RAG 有什么区别？
系统:
  1. 先检索课程资料
  2. 找到 Week6 LoRA 和 Week8 RAG 内容
  3. 组织对比答案

用户: 再给我按“训练时”和“推理时”重新比较
系统:
  4. 利用 history 理解“再比较”的指代
  5. 结合上轮上下文继续回答
```

## Part 3：系统架构设计

### 3.1 最小可行架构

```text
用户输入
  ↓
会话管理器
  - 保存 history
  - 控制上下文预算
  ↓
Agent 控制器
  - 判断是否需要检索
  - 判断是否需要工具调用
  ↓
RAG 检索器
  - query 改写
  - top-k 召回
  - 重排/筛选
  ↓
工具层
  - 搜索
  - 计算器
  - 文档读取
  ↓
LLM 生成器
  - 生成最终答案
  ↓
答案返回 + 写回会话状态
```

### 3.2 推荐模块拆分

建议拆成以下几个类：

```python
ChatSessionManager
ToolRegistry
SimpleReActAgent
SimpleRetriever
RAGPipeline
KnowledgeAgentApp
```

这样可以避免把所有逻辑都塞进一个函数里。

## Part 4：多轮对话、Agent、RAG 各负责什么

### 4.1 多轮对话层

负责：

- history 保存
- 截断与压缩
- 用户上下文承接

### 4.2 Agent 层

负责：

- 决定是否要行动
- 决定调用哪个工具
- 组织多步执行流程

### 4.3 RAG 层

负责：

- 提供外部知识
- 让回答有依据
- 提高知识时效性和可控性

三者的关系可以理解为：

```text
多轮对话 = 会话基础设施
Agent = 决策与行动层
RAG = 外部知识访问层
```

## Part 5：一个典型请求是如何流动的

### 5.1 数据流示例

```text
用户: QLoRA 为什么能省显存？

Step 1: SessionManager
  读取最近几轮 history

Step 2: Agent
  判断该问题需要查资料，而不是只靠内部记忆

Step 3: Retriever
  检索 “QLoRA 显存 节省 原理”
  找到 NF4 / 双重量化 / 分页优化器 相关 chunk

Step 4: Generator
  基于 chunk 组织答案

Step 5: SessionManager
  把回答写回 history
```

### 5.2 什么时候需要“工具”而不是“检索”

适合检索：

- 查课程内容
- 查知识库文档
- 查历史笔记

适合工具调用：

- 计算数值
- 获取实时天气
- 调用 API
- 操作数据库

如果任务需要两者结合，Agent 就要负责编排。

## Part 6：如何评估这个综合系统

### 6.1 不要只看“能不能回答”

综合项目至少应从四个维度评估：

1. 检索质量
2. 回答质量
3. 多轮连贯性
4. 工具调用可靠性

### 6.2 建议的评估问题

#### 检索评估

- 给定问题，top-k 是否包含正确 chunk？
- 检索结果里噪声是否过多？

#### 回答评估

- 是否基于检索材料作答？
- 是否出现明显幻觉？
- 是否能承认资料不足？

#### 多轮评估

- 能否理解“它”“刚才那个方法”等指代？
- 上下文截断后是否还能保持一致性？

#### 工具评估

- 是否选择了正确工具？
- 参数是否解析正确？
- 工具失败时是否能优雅降级？

### 6.3 一份最小评测表

| 维度 | 指标 | 现象 |
|------|------|------|
| 检索 | top-k 命中率 | 是否召回正确 chunk |
| 回答 | groundedness | 是否基于证据作答 |
| 多轮 | context consistency | 是否记住上文约束 |
| 工具 | tool success rate | 工具调用是否成功 |

## Part 7：常见失败模式与排查思路

### 7.1 用户问得很清楚，但回答仍然偏题

优先检查：

- 检索结果是否正确
- chunk 是否太长
- prompt 是否把无关上下文也塞进去了

### 7.2 系统总是调用不必要的工具

优先检查：

- tool description 是否写得太宽泛
- prompt 是否鼓励过度调用
- 有没有设置 max_tool_calls

### 7.3 多轮问答到后面开始遗忘

优先检查：

- history 截断是否过激
- system prompt 是否稳定保留
- 是否需要摘要压缩或长期记忆

### 7.4 检索到了正确 chunk，但模型还是乱答

优先检查：

- 是否做了 prompt 约束
- 检索结果顺序是否合理
- chunk 是否被噪声稀释
